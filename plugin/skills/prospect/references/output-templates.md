## Tier-A outreach pack (§7.5) — `prospects-YYYYMMDD-outreach-[company-slug].md`

Generate one for **every** Tier-A (🔥) account. **Read the voice file first** — `profiles/<active>/knowledge/voice.md` — every line is checked against it. These are **drafts**; the colleague reviews and sends manually. Never auto-send.

````markdown
# Outreach Pack — [Company] — [Date]

**Tier:** A 🔥 | **Score:** X/Y (fit + heat) | **Segment:** Enterprise/Startup | **Market:** [geo]
**Primary persona:** [Name] — [Title] [🆕 if new-in-role]
**Why-now (🔥):** [exact signal used as hook — date + source URL]
**Heat / timing:** [heat +n — feed(s); 🆕 new-in-role persona + start month / — . Internal context only — no outreach line references it.]
**Mapped case study:** [from `profiles/<active>/knowledge/case-studies.md`]
**Gift artifact:** [the give-first asset touch 1 offers and touch 2 delivers — teaser one-pager / demo mockup screenshot / custom audit; file path or one-line spec]

---
## LinkedIn DM (≤ 280 chars)
> [Signal-first. One bridge. One ask. No fluff.]

**Character count:** [n] / 280

---
## Email (touch 1)
**Subject:** [1–4 words, lowercase, names their signal — reads like an internal note]
> [Opening: name the signal precisely, with one clause of credit — their move is right.]
>
> [Gap: the problem in their vocabulary — a missing step in their system. No product pitch, no feature list; the artifact carries the product after they reply.]
>
> [Proof: the mapped case study — company + what they shipped. First cut if over length.]
>
> [Ask: one offer-CTA gating the gift artifact — "want the mockup?" Never a time-ask or calendar link.]
>
> [signature from PROFILE]

**Word count:** [n] / 100 — plain text; no images, attachments, or calendar links; ≤1 untracked link

---
## Micro-deck outline (3 slides — for when they say yes)
**Slide 1 — Their problem (named):** their situation as a problem statement; 2–3 bullets on the gap they have now per the why-now signal.
**Slide 2 — The case study (matched):** how [case-study company] solved the same shape; 3 outcome bullets (what shipped, how fast, what changed).
**Slide 3 — One clear next step:** "We'll show you [the lead-product capability] in your stack — [specific outcome]. No deck, just a live demo." + CTA.
*(Build it later with `build-deck`, Phase 3/4. The outline stands alone as talk-track.)*

---
## Sequence — 5 touches / 2 channels over ~12 days (each adds something new; never "just checking in")

Touch 1 is the email above; the LinkedIn DM doubles as the Day-2 connect note. On **any reply**, stop the clock and deliver the gift artifact (that's the offer working). Draft every touch **now** — follow-through must cost nothing later.

**Touch 2 — Day 2, LinkedIn:** connection request using the DM above (or a trimmed variant ≤200 chars). No pitch, no link.

**Touch 3 — Day 5, email (same thread):** different-angle insight, same signal — new observation, peer example, or threat-model note. Re-offer the artifact in different words.
> [draft]

**Touch 4 — Day 8, LinkedIn:** substantive comment on their recent post (no pitch) — or, if connected and they've posted nothing, a 1–2 line DM with one specific observation.
> [draft]

**Touch 5 — Day 12, email (new thread + new subject):** breakup that **gifts the artifact anyway** — attach/link it, no ask, one line on what it shows. Then park the account on the nurture list (monthly no-ask value touch); a fresh signal re-opens it.
> [draft]

---
## Threading — role-differentiated first lines (4–6 personas per Tier-A account)

Same signal, same artifact — a different first line per persona (their hook-matrix row × this account's 🔥 column). Touch 1 goes to the primary; stagger the others 1–2 days behind. One row per additional persona:

| Economic buyer | | | email / LinkedIn |
| Co-signer / Technical evaluator | | | |
| Influencer | | | |

---
## Notes / risks for [colleague]
[Anything affecting the send — recent competitor partnership, press, sensitivity around the signal, suggested timing.]
````

## Outreach log (cross-run rollup)

`content/<active>/prospects/outreach-log.{md,csv}` — every Tier-A outreach pack ever drafted for the profile, one row each: date, account, tier, persona, verified email (or "to enrich"), email subject, channels, path back to the full pack. Built by `python -m gtm_core.outreach_log build --profile <active>` (see SKILL.md Step 7) by parsing the packs already on disk — it is a derived view, not a separate write path, so it never drifts from the account folders. Regenerate after every run, including runs with zero Tier-A accounts (a no-op rewrite is fine). This is the answer to "what have we drafted so far" — read it instead of grepping account folders by hand.

## Per-run QA checklist (§8) — before saving

- [ ] Run mix matches the PROFILE segment_mix (default 3 enterprise + 7 startup), split across target markets (or documented reallocation in header)
- [ ] All accounts score at/above threshold (enterprise ≥6/12; startup ≥6/10)
- [ ] ≥3 accounts at Tier A (enterprise ≥8/12; startup ≥7/10) — if fewer, note "broaden signal search next run"
- [ ] No account triggers any Gate-B disqualifier for its segment
- [ ] Every account has ≥1 contact on a champion/primary-buyer persona (verified email on Vibe runs; unverified flagged on web-only runs)
- [ ] Every account has ≥1 dated "why now" signal mapped to an ICP trigger
- [ ] Every account has a hook-matrix hook selected (not free-form) and a case study mapped — 🆕 accounts use the new-in-role column
- [ ] No account inside the trailing 60-day re-show window (see SKILL.md §exclude)
- [ ] Tier-A flagged 🔥 in header and section titles; Tier-A outreach packs generated
- [ ] Heat axis recorded per account (`heat` 0–3 + `intent_feeds` + 🆕 where found); boosted totals capped at the rubric ceiling
- [ ] Tier-A section order = heat → 🆕 new-in-role → signal recency (the queue ordering)
- [ ] Every Tier-A pack has all 5 touches drafted (2 channels) + a threading table with ≥4 personas and first lines
- [ ] No outreach line quotes or implies intent-feed data (public signals only in copy)
- [ ] Fit-but-cold accounts listed under "Nurture" (no meeting-ask sequence)
- [ ] `rubric_version` present; metered spend logged in header (≤ per-run cap, or "0 — web-only")
- [ ] HubSpot CSV written
- [ ] Outreach log regenerated (`python -m gtm_core.outreach_log build --profile <active>`)
